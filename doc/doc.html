Introduction

Notre application se décompose en 2 parties. La première partie est un script python qui permet de générer des tables de hachages au format json à partir d'un fichier de texte. La seconde partie est une page web qui permet d'effectuer différent traitement sur ces tables de hachages avec les fichiers json.

Par défaut, l'application fournit déjà les fichiers json et la page web peut être utilisé telle qu'elle. Cepandant si vous souhaitez, ajouter de nouvelle langue ou modifier le dossier d'exportation des fichier json, vous pouvez utiliser le script python.

Attention ! Il est impératif de lancer le script avec le navigateur Firefox. En effet tous les autres navigateurs ont bloqué la possibilité de pouvoir lire des fichiers sur le disque de l'utilisateur. La page peut également être lancé sous Chrome si on lance Google Chrome avec le paramétre suivant chrome.exe --allow-file-access-from-files. Il est également impératif de lancer la page index.html au même niveau que le dossier hash sinon le script ne va pas fonctionner et provoquer une erreur.

Script python

Le script python se nomme "hashtable.py" il permet de pouvoir générer les différentes table de hachage dans un fichier en format json. Ces fichiers json seront utilisé par la page web. Le script python va prendre les fichiers .txt contenu dans un dossier pour générer les tables de hachage. Notre script peut être executé avec différents paramètres :

  • Le première paramètre corresponds au chemin du dossier source. Le script va lire tous les fichiers .txt du dossier afin de générer les tables de hachages.
  • Le deuxième paramètre corresponds au chemin du dossier de destination. C'est ici que seront exportés les tables de hachages au format json.
  • Si le script est executé sans aucun paramètre, le chemin du dossier source est par défaut dict et le dossier d'exportation hash.

Voici un exemple d'éxecution du script : >>python hashtable.py souce destination. Dans cet exemple, le script python va lire les fichiers .txt dans le dossier source et les tables de hachages seront exporté dans export

Attention ! Le script va d'abord supprimer le dossier d'exportation, vérifiez donc à utiliser un nom de dossier qui n'existe pas encore, ou que le dossier déjà existant n'est plus utile.

Application de détection

Au niveau de l'application de détection, son utilisation est très simple.

Tout d'abord nous avons un champs pour entrer du texte. Il suffit d'entrer une phrase de plusieurs mots, pour que notre algorithme détecte la langue dans laquelle la phrase est écrite. Une fois la langue détectée, un petit drapeau représentant la langue sera affiché en dessus.

On peut également voir la probabilité que le texte sois écrit dans chacunes des langues.

En dessous du drapeau, il est également affiché le pourcentage que le texte écrit soit bien dans la langue détectée.

Au niveau de la correction, les erreurs sont affichées dans le textarea. En effet, les mots n'étant pas contenus dans le dictionnaire de la langue détectée sont surlignés en rouge.

A quoi sert la liste déroulante en bas ? En sélectionnant une langue, des informations pour le dictionnaire de mots correspondant seront affichées. Ces données sont :

  • Number of words : correspond au nombre de mots présents dans le dictionnaire
  • Average time access : le temps d'accès moyen à un mot dans la table
  • Number of collisions : le nombre de collisions présentes dans la table de hachage
  • Filling rate of the HashTable : le taux de remplissage de notre table de hachage
